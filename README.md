# Doorman Secure

**Secure clinical message routing. Engineered for healthcare environments.**

Security and trust are not features — they are structural requirements.  
Doorman Secure is built to reduce risk in clinical communication systems through verifiable cryptography, strict access control, and immutable audit trails.

---

## The Mission

Hospitals and clinics move lives — not just data.

Legacy systems introduce operational and compliance risk.  
Doorman Secure is designed to reduce that risk surface with clear, enforceable security guarantees.

---

## Core Capabilities

- Cryptographically verified webhook routing (HMAC-SHA256)
- AES-256 encryption at rest and in transit
- Stateless authentication via JWT
- OAuth2 identity federation
- Service-to-service authentication (API keys)
- Row-Level Security (RLS) for least-privilege data access
- Immutable audit logging (365+ day retention)
- Designed for automation and AI integration

---

## Security Principles

**Verified by default**  
Every request must be authenticated and validated.

**Least privilege architecture**  
Access is scoped, measurable, and controlled.

**Traceable operations**  
All system activity is logged and auditable.

**Patient-first engineering**  
Clinical safety precedes feature velocity.
